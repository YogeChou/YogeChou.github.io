Elevator Simulator

Simulator for Passengers Moving and Elevators Dispatching

This system simulates elevator dispatching logic, movement scenarios, and passenger boarding behaviors to expedite the validation of various applications within elevators. As elevator configurations vary across different environments and passengers exhibit diverse boarding behaviors, it is crucial to tailor design strategies to each location for maximal waste reduction during the boarding process.

Given that a single run of the simulation program takes 5 to 10 minutes, exhaustive testing of the myriad strategies is impractical. Therefore, we employ a genetic algorithm to accelerate the discovery of optimal solutions through continuous iteration and crossover, aiming for an acceptable local optimum, if not a global one.

Occasionally, we also utilize AWS Batch for parallel computation. While incurring certain costs, this approach significantly enhances computational speed.

Smart Elevator Kit

AI solutions for ALL elevators

SEK is a plug-in smart device installed on elevators that utilizes cameras, microphones, and other sensors to perceive the current status of the elevator. It can report abnormal behaviors within the cabin, support voice-activated calling, and achieve intelligent dispatching—applications that are challenging for traditional elevators to implement.

The backend development of SEK encompasses various aspects:

  • OTA System: Due to the high cost of performing software updates on-site, the functionality of remote updates is crucial infrastructure for the Internet of Things (IoT).
  • Device Communication Module: Whether between software modules within a single device or among different devices, there is a need for information exchange. Therefore, providing a universal and stable communication interface is another essential infrastructure.
  • Real-time Reporting Platform for Elevator Status: Maintaining real-time status for each elevator in the cloud, including floor, direction of movement, door status, etc. This platform supports subsequent app and backend information presentation, and even usage by third-party vendors for robot calling services.
  • API Development: Providing APIs for internal management backend, owner backend, and app usage.
  • DevOps: This includes cloud DevOps and OTA DevOps.
    • Cloud DevOps automates the construction of API Gateway + Lambda using AWS CDK, including Lambda code deployment and linking Lambda with other resources.
    • OTA DevOps utilizes a pipeline to automatically generate software versions needed for the device, deploying them to corresponding machines based on branches.

eKeeper

IOT Solution for Elevator Controller

【eKeeper】is an elevator Internet of Things (IoT) project that involves transmitting elevator signals to the cloud. This allows elevator operators to intelligently manage a large number of devices distributed across different locations. In addition to obtaining real-time information about the current status of elevators, the ability to remotely locate issues during elevator malfunctions enhances the efficiency of maintenance processes. Monitoring the wear and tear of components enables more precise maintenance, effectively preventing unexpected incidents.

The backend development of eKeeper encompasses various technologies:

  • ETL (Extract, Transform, Load): Processing large volumes of signal data collected from elevators and sending it to a data lake for subsequent analysis.
  • Data Storage: Managing elevator signal data and user data from the website's backend, designing different storage strategies based on storage costs and usage scenarios.
  • API Development: Creating APIs for use by the website and app.
  • Cloud Resource Deployment: Setting up and deploying API servers and DB servers, along with the deployment of ETL data flows, data storage, and more.
